# Systems Engineering & Simulation Labs

> **Status:** Phase 1 (Foundation Hardening)  
> **Track:** NUCLEAR 6M (May - Oct 2026)  
> **Mission:** Documenting the technical "Core Thread" and Systems/Networking experiments.

This repository serves as an **Artifact-First Proof of Mastery**. Every directory contains runnable code, benchmarks, or technical deconstructions of fundamental engineering concepts.

## 🛠️ Lab Structure

| Module | Focus | Output Type |
|---|---|---|
| [`/sql`](./sql) | Join logic, CTEs, Window functions, Query Optimization | `.sql` drills + performance notes |
| [`/systems`](./systems) | OS internals, memory management, process models, profiling | `.py` scripts + profiling logs |
| [`/networking`](./networking) | TCP/UDP, WebSockets, Load Balancing, Network Resilience | Capture logs + experiment postmortems |
| [`/math-sim`](./math-sim) | NumPy vectorization, Physics integrators, Simulation engines | Benchmarks + numerical engines |
| [`/notes`](./notes) | First-principles technical deconstructions | Markdown "Whiteboard" notes |

## 📜 Rules of the Lab

1. **Proof-Driven:** No concept is "learned" until there is a runnable experiment or a benchmark.
2. **Benchmark Everything:** If it's code, measure its latency, memory, or throughput.
3. **Whiteboard Ready:** Every note in `/notes` must be clear enough to explain on a whiteboard in 5 minutes.
4. **No-AI Core:** The "Core Thread" (SQL/Math/OOP) is solved manually to ensure deep retention.
